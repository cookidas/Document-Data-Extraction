# 🌾 화성시 농업 직불금 자동화 시스템

고령 농업인을 위한 **직불금 셀프 신청 웹앱**과 담당 공무원을 위한 **OCR 검수 대시보드**를
하나의 데이터 흐름으로 통합한 프로토타입입니다.

> 제1회 화성 AI·DATA 기반 솔루션 경진대회 출품작 (분석과제 3: 공공 마이데이터 기반 직불금 신청 시스템 구축)

## 왜 만들었나

- 담당 주무관 1인이 하루 수백 건의 수기 신청서를 컴퓨터에 재입력하는 **이중 업무**가 발생합니다.
- 국가 신청 앱(농업e지)이 있지만 절차가 복잡해 고령 농업인은 결국 **행정복지센터를 방문**합니다.
- 이 프로젝트는 어르신의 행동(방문)은 그대로 두고 **뒷단의 데이터 흐름만 디지털화**합니다.
  창구 태블릿에서 담당자와 함께 이 웹앱으로 접수하면, 종이 신청서 재입력이 사라지고
  다음 해에는 어르신이 집에서 혼자 재신청할 수 있게 됩니다.

```
신청자(모바일 웹) ──> Flask + SQLite ──> 담당자(검수 대시보드)
  사실확인서 촬영        공유 DB             신뢰도 하이라이트
  CLOVA OCR 자동입력    이미지 파일 저장      크롭 돋보기 · 비교 검수
```

## 주요 기능

### 신청자 웹앱 (`/citizen/apply.html`)
- **토스(Toss) 스타일 UX**: 한 화면에 하나의 질문, 진행바, 큰 글씨·큰 버튼, TTS 음성 안내
- **PASS 스타일 본인인증** (시연용 Mock — 실제 연동 아님)
- **신청 유형 3분기**
  - 처음 신청해요 → 담당자 코드 입력 후 창구 태블릿에서 직원 동반 접수 (규정상 첫 신청은 방문 필수)
  - 이전에도 신청했어요 → 서류 없이 주민번호 조회만으로 접수 완료 (변경없음 재신청)
  - 이전 신청 정보가 바뀌었어요 → 서류 재제출 (변경있음 재신청)
- **농업경영체 사실확인서 촬영 → CLOVA OCR 자동 입력** (사진·PDF 모두 지원, 2쪽 PDF는 쪽 자동 추출)
- 인식 결과를 2~3개씩 묶어 확인 (서류를 건너뛰면 직접 입력 모드로 문구 자동 전환)
- **직불금 자동 비교**: 소농직불금(정액 130만원) vs 면적직불금(누진 단가)을 자동 계산해
  더 유리한 쪽을 큰 글씨로 추천

### 담당자 대시보드 (`/staff/`)
- 접수유형 4분류 배지: 🏢 현장 종이접수 · 🧑‍💼 현장 도움접수 · 🔄 재신청(변경있음) · ♻️ 재신청(변경없음)
- OCR **신뢰도 기반 색상 하이라이트** (90%↑ 초록 / 80~90% 노랑 / 80%↓ 빨강 — 체크 후에만 저장 가능)
- 입력창에 마우스를 올리면 원본 사진의 해당 영역을 확대해 보여주는 **크롭 돋보기**
- 재신청(변경있음)은 **비교 검수 모드**: 좌측 이전 데이터 / 우측 현재 데이터, 달라진 필드 점선+🔄 표시
- 소농/면적 직불금 자격 자동 판정 + 예상 지급액 표시, 검수 완료 건 CSV/Excel 내보내기 (주민번호 마스킹)

## 실행 방법

```bash
pip install -r requirements.txt
python server.py
```

| 주소 | 화면 |
|---|---|
| http://localhost:5000/citizen/apply.html | 신청자 웹앱 (모바일 최적화) |
| http://localhost:5000/staff/index.html | 담당자 로그인 |
| http://localhost:5000/staff/completed.html | 검수 완료 목록 |

**데모 계정/코드** (프로토타입 하드코딩)
- 담당자 로그인: `admin` / `admin1234`
- 현장 도움접수 담당자 코드: `2026` (`shared/js/config.js`의 `STAFF_ASSIST_PIN`)

**CLOVA OCR 연결 (선택)**
- NAVER CLOVA OCR Custom 도메인의 Secret Key를 담당자 대시보드 우측 상단 ⚙️에서 입력하면
  실제 OCR이 동작합니다 (키는 브라우저 localStorage에만 저장, 서버는 프록시만 수행).
- 키가 없어도 전체 흐름은 동작합니다 — OCR 값 없이 직접 입력 모드로 진행됩니다.
- 템플릿: 직불금 신청서 1·2장(창구 스캔용), 농업경영체 사실확인서 1·2쪽(셀프접수용).
  자신의 CLOVA 도메인을 쓰려면 `api/ocr.py`의 invoke URL과 `shared/js/config.js`의
  `CLOVA_TEMPLATE_IDS`를 교체하세요.

폰으로 테스트하려면 PC와 같은 네트워크에서 `http://<PC IP>:5000/citizen/apply.html`로 접속하거나,
`cloudflared tunnel --url http://localhost:5000` 같은 터널을 사용하세요.

## 프로젝트 구조

```
server.py            Flask 앱 (정적 서빙 + API 블루프린트 등록)
db/                  SQLite 스키마·리포지토리 (direct_payment.db 자동 생성)
api/                 REST API (applications CRUD·이미지 업로드/서빙·CLOVA 프록시)
uploads/             신청서 이미지 파일 (신청서 ID별 폴더)
staff/               담당자 화면 (로그인·대시보드·검수·완료목록)
citizen/             신청자 화면 (단계별 마법사: intake→capture→eligibility→finish)
shared/              공용 모듈 (config·OCR 클라이언트·직불금 판정 로직·API 클라이언트)
```

## 직불금 판정 로직 (2026년 기본형 공익직불금 기준)

- **소농직불금** (정액 1,300,000원): 본인 경작면적 0.1~0.5ha, 가구 전체 소유면적 1.55ha 미만,
  영농 종사·농촌 거주 각 3년 이상, 가구 종합소득 4,500만원 미만(자기신고) — 모두 충족 시
- **면적직불금**: 논/밭 각각 구간별 누진 단가 적용 후 합산
  (논: 2ha까지 187만/ha, 6ha까지 179만, 초과 170만 · 밭: 150만/143만/136만)
- 사실확인서 2쪽의 필지 표에서 소유자가 "본인"인 행의 합 = 신청면적, 전체 행의 합 = 가구 소유면적

## 한계와 주의사항 (프로토타입)

- PASS 본인인증은 **UI만 구현한 Mock**입니다 (사업자 계약 없이 실연동 불가).
- 마이데이터 연동은 제도적으로 불가하여 사실확인서 OCR로 대체했습니다.
- 인증·세션·암호화가 프로토타입 수준입니다. 실서비스 전환 시 HTTPS, 서버사이드 암호화 저장,
  개인정보 처리 위탁계약, 실제 본인인증 연동이 필요합니다.
- `direct_payment.db`와 `uploads/`에는 개인정보가 저장될 수 있으므로 저장소에 커밋하지 마세요
  (`.gitignore`에 포함되어 있습니다).

## 개발 정보

- 기술 스택: Vanilla JS/HTML/CSS · Python Flask · SQLite · NAVER CLOVA OCR (Template) · SheetJS · pdf.js
- 본 프로젝트는 생성형 AI(Claude)를 활용해 개발되었으며, 설계·검증·수정은 팀이 직접 수행했습니다.
